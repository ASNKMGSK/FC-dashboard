제조 현장을 위한 ML 기반 AI 운영 시스템

H형강 사상압연 공정의 3개 라인 x 9개 스탠드를 실시간으로 모니터링, 제어, 분석하는 풀스택 AI 플랫폼.
7종 ML 모델과 Canvas 60fps 실시간 차트로 설비 진단, 품질 SPC, 공정 최적화, 예지보전을 수행합니다.

핵심 기능

6개 운영 패널로 사상압연 공정의 전 라이프사이클을 커버합니다. 각 패널은 dynamic() import로 필요 시에만 로드되어 초기 번들 크기를 최소화합니다.

1. FMCS 실시간 모니터링 (StandControlPanel)

FMCS 사상압연 라인의 9개 스탠드를 실시간 제어하는 핵심 패널입니다. 백엔드의 StandSimulator 엔진이 시간 기반 연속 시뮬레이션으로 물리적으로 일관된 공정 데이터를 생성합니다.

  • 빌렛 이동 애니메이션: 9 스탠드 순차 진행 스텝 인디케이터, 현재 스탠드 펄스 애니메이션
  • AI 자동제어 / 수동제어 모드 전환 (ai_auto / manual)
  • 스탠드별 전류/속도/하중/온도: Canvas 2D 기반 60fps 실시간 차트 (RealtimeLineCanvas)
  • Cascade Mode: 연동 제어 ON/OFF, Roll Gap, HMD 상태 표시
  • 스탠드 상세 패널: 클릭 시 Load/Speed 상세 차트 + 운전 파라미터
  • 3개 라인 선택: FM-LINE1, FM-LINE2, FM-LINE3 (각각 독립 시뮬레이터 인스턴스)

2. 품질 관리 / SPC 분석 (AnalysisPanel)

공정 품질 데이터를 5개 서브 탭으로 다각도 분석합니다.

  • TrendTab: 시계열 추이, SPC 관리도 (UCL/LCL)
  • AnomalyTab: Isolation Forest 기반 이상 패턴 탐지, 피스별 SPC
  • PredictionTab: 수율/생산량/OEE 예측 시뮬레이션
  • MaintenanceTab: 정비 이력, 자동배정, 품질평가 점수
  • LifecycleTab: 설비 잔존수명(RUL) 예측, 교체 주기 분석

3. AI 예측 / MLOps (ModelsPanel)

7종 ML 모델의 전체 라이프사이클을 통합 UI에서 관리합니다.

  • MLflow 모델 레지스트리: 모델별 버전 관리 UI (Production/Archived), 배포 이력 추적
  • Optuna 하이퍼파라미터 자동 튜닝: 베이지안 최적화로 재학습 시 최적 파라미터 자동 탐색
  • AutoGluon 앙상블: 다중 알고리즘 자동 앙상블, 5~10% 성능 개선
  • 모델 드리프트 모니터링: RMSE 30일 추이 + Feature PSI (임계값 0.003)
  • 원클릭 재학습: SSE 스트림 4단계 진행률 (전처리 → 학습 → 평가 → 배포)
  • A/B 테스트: RMSE, R2, MAE, 추론시간(ms) 비교 → 자동 추천

4. 정비 관리

예지보전(Predictive Maintenance) 기반 자동화 시스템입니다.

  • 위험 설비 자동 탐지: SHAP 기반 고장 위험 요인 Top-5, Heuristic + ML 스코어링
  • 자동 정비 계획: 설비 분석 → 위험도 평가 → 정비 방안 → 조치 실행
  • 트러블슈팅 FAQ: TF-IDF + KMeans → Elbow Method → LLM 가이드 생성
  • 생산 리포트: 일별/주별/월별 KPI 자동 집계 + 이상 이벤트 하이라이트

기술 아키텍처

3-Tier 프록시 아키텍처: Browser → Next.js :3000 → FastAPI :8001. 브라우저는 Next.js만 바라보고, Next.js가 백엔드를 프록시하여 CORS/인증을 통합 관리합니다.

데이터 흐름

state.py가 전체 시스템의 중심입니다. 모든 모듈이 import state as st로 참조하는 싱글톤 패턴으로 DataFrame, ML 모델, 설정을 한 곳에서 관리합니다.

  • 시작: lifespan에서 init_data_models() → PKL 로딩 + ML 모델 로딩 (ThreadPoolExecutor 병렬)
  • 최적화: _optimize_df_memory() — object→category, int64→int32, float64→float32
  • 시뮬레이터: StandSimulator 싱글턴, 시간 기반 연속 데이터 생성
  • 캐시: EQUIPMENT_PERF_MAP 딕셔너리로 O(1) 설비 성능 조회

성과 / 차별점

분절된 공정 관리 도구를 AI 통합 플랫폼으로 대체. LLM + ML 결합으로 운전자 의사결정 시간을 획기적으로 단축합니다.

주요 성과

  • 7종 ML 모델 통합 운영: 분류, 회귀, 이상탐지, 클러스터링을 하나의 플랫폼에서 학습/배포/모니터링
  • 60fps 실시간 모니터링: 9 스탠드 x 4채널 = 36개 스트림, Canvas 기반 시각화
  • 자연어 설비 진단: "EQ0001 진단해줘" 한 마디로 분석 → 예측 → 정비 계획 자동 실행
  • E2E 자동화 예지보전: 탐지 → SHAP 분석 → 정비 계획 → 조치의 완전 자동화
  • SSE 실시간 UX: 재학습/예지보전/채팅 모두 SSE 스트리밍으로 즉각적 피드백

차별점

1. LLM + ML 하이브리드 AI

LLM의 자연어 이해 + ML의 예측 정확성. 자연어 질의 → 의도 파악 → ML 도구 호출 → 데이터 기반 분석 결과를 자연어로 설명합니다.

2. 물리 기반 시뮬레이션 엔진

Sin 룩업 테이블(1024 포인트), 시간 기반 연속 시뮬레이션으로 실제 사상압연 공정의 물리적 특성을 재현합니다. 스탠드 진행에 따른 전류 증가, 온도 감소 패턴을 반영합니다.

3. Dual-Layer 에이전트 도구 패턴

tool_*() 구현 + @tool 래퍼 분리로 동일 로직을 API와 LLM에서 중복 없이 재사용합니다.

4. Canvas 60fps 차트 엔진

통합 RAF, Gradient 캐싱, Binary Search Culling 등 6가지 최적화로 대규모 스트림에서도 안정 60fps.

5. E2E 자동화 파이프라인

SHAP + KMeans + LLM 조합으로 운전자 개입 없는 완전 자동화 예지보전/트러블슈팅/리포트 워크플로우.

6. 프로덕션 레벨 최적화

  • DataFrame dtype 자동 최적화 (메모리 30~50% 절감)
  • ML 모델 Lazy Loading
  • ThreadPoolExecutor 병렬 데이터 로딩
  • Page Visibility API 비활성 탭 폴링 중단
  • dynamic() import 코드 분할 (SSR: false)
  • GZip 응답 압축
